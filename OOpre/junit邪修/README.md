# Junit 邪修指北

## 写在前面：

Junit测试本身十分重要，尤其是在日后的工作中，当代码产出后程序员需要做的工作就是不断地测试，大家还是应该按照课程组的规定，认真的完成几版junit测试文件的编写的。本文只适用于前期**救急**以及后期java文件的码量起来之后的**节约时间**😁。

## 开发过程中可能会遇到的情况：

- 有些同学开发习惯较好，代码鲁棒性高，总会在可能越界，空指针的地方加以判断，这就会导致在数据点不强时分支覆盖率和行覆盖率难以达到课程组要求。
- 还有的同学就是纯懒的。

相信大家在oopre过程中发愁的不仅是测试点，更多的其实在于junit测试文件。笔者在这里给出一个Junit邪修小技巧，希望大家适度使用。

## 使用步骤

1. 提交一版作业答案，此时可以不管junit
2. 下载**中测进阶**样例，命名`std_in.txt`
3. 运行`transform.py`,生成`trans_to_info.txt`
4. 将`trans_to_info.txt`的内容复制到`InputTest.java`的相应位置。

具体可以参照笔者给出的示例。
